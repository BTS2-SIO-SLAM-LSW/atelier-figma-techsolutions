## 📝 Cas d'utilisation avec Personas

### Définition
Format standard : **"En tant que [persona], je veux [action] pour [objectif]"**

### Exemples avec justifications
- **Thomas (Senior)** : "Voir tickets urgents en ≤ 2 clics" *(4 clics actuellement = friction)*
- **Sophie (Junior)** : "Créer ticket sans oublier urgence" *(25% d'erreurs actuelles)*
- **Marie (Manager)** : "Vue d'ensemble équipe temps réel" *(dispersion actuelle)*
- **Julien (Itinérant)** : "Interface tablette ergonomique" *(texte trop petit)*

### Diagramme enrichi
```
[Thomas-Senior] ──── (🔴 Urgences ≤2 clics) ──── (📊 Vue équipe)
       │                    │                        │
[Sophie-Junior] ──── (✏️ Création guidée) ──── (❓ Aide contextuelle)
       │                    │                        │
[Marie-Manager] ──── (📈 Dashboard KPI) ──── (👥 Répartition charge)
       │                    │                        │
[Julien-Mobile] ──── (📱 Interface tactile) ──── (📷 Photos terrain)
```

## 📱 Bonnes Pratiques UX/UI (Nouveau Guide)

### 🧠 **Lois UX appliquées**
- **Loi de Fitts** : Urgences grandes et accessibles (≤ 2 clics)
- **Loi de Hick** : 4 champs au lieu de 12 (réduction choix)
- **Loi de Miller** : 7±2 éléments par groupe (charge cognitive)

### 🎨 **Design Patterns**
- **Hiérarchie couleurs** : 🔴 Critique → 🟠 Haute → 🟡 Moyenne → 🟢 Basse
- **États composants** : Normal, Hover, Active, Disabled, Loading
- **Responsive design** : Mobile-first, breakpoints 768px/1024px

### 🔧 **Figma Pro**
- **Design System** : Tokens de couleurs, composants, variants
- **Auto Layout** : Responsive automatique
- **Prototypage avancé** : Smart Animate, overlays

### ♿ **Accessibilité WCAG 2.1 AA**
- Contraste ≥ 4.5:1 • Boutons ≥ 44px • Navigation clavier
